## Vibethon official requirements (condensed)

### Problem statement
- Build an **interactive web-based AIML learning platform** that is engaging, practical, and accessible.
- Must go beyond static content: **learn concepts**, **practice hands-on**, **experiment with real scenarios**, **gamified engagement**.
- Goal: simplify complex AIML while keeping **technical depth**, enabling **learn + apply + explore** in one platform.

### Challenge constraints
- **Rapid prototyping**: functional prototype within **4 hours**.
- Prioritize **execution quality over feature quantity**.
- Demonstrate structured thinking and efficient use of AI tools.

### Expected capabilities (pick multiple; not all mandatory)
- **Auth**: email/password login & registration; track basic activity/progress; OAuth optional.
- **Learning modules**: levels (Beginner → Intermediate → Advanced) with explanation, examples/use-cases, optional visuals.
- **Interactive coding playground**: in-browser (preferably Python) run/modify sample scripts; sandbox/Jupyter optional.
- **Mini-games**: simple interactive visual games for concepts (decision trees, classification, NN basics).
- **Quizzes**: module-wise MCQs + basic logic/code questions; instant feedback + scoring.
- **Simulations**: small real-world AIML use cases (spam detection, image classification demo) with datasets + interaction flow.
- **Progress dashboard**: completed modules, scores, activity; progress bars/levels.
- **Leaderboard & gamification**: ranks, badges, scores, streaks.
- **Responsive design**: desktop/tablet/mobile.
- **Deployment & OSS**: hosted if possible or runnable locally; repo includes README, setup, screenshots/demo references.

### Mandatory minimum criteria (Round 1)
- **Working prototype** (not only UI screens).
- **At least one structured learning module** implemented.
- **At least one interactive feature**: quiz / game / coding / simulation.
- **Proper GitHub repo activity** with scheduled commits (per rulebook).
- Must be able to **demo** during evaluation.

### Shortlisting factors
- Working prototype
- GitHub activity
- Feature implementation quality
- Overall execution

### “Win conditions” (what to optimize)
- A polished, stable end-to-end flow (onboarding → learn → interact → score → progress).
- Clear mapping from features to requirements, with a short demo script.
